== 11. CI/CD Implementation with OpenShift Pipelines (Tekton)

=== 11.1 CI/CD Architecture Overview

Implemented a Kubernetes-native CI/CD workflow using OpenShift Pipelines (Tekton) to automate:

* Source code retrieval from Git repository
* Container image build
* Image push to internal registry
* Automated deployment rollout via ImageStream trigger

Pipeline Flow:

Git Push → PipelineRun → Build Task → Push Image → ImageStream Update → Deployment Rollout

---

=== 11.2 Pipeline Components

Core Tekton Resources:

* Task
* Pipeline
* PipelineRun
* Workspace
* ServiceAccount

Example: Pipeline Definition

[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: jhipster-ci-pipeline
spec:
  workspaces:
    - name: shared-workspace
  tasks:
    - name: clone-repo
      taskRef:
        name: git-clone
      workspaces:
        - name: output
          workspace: shared-workspace

    - name: build-image
      runAfter: ["clone-repo"]
      taskRef:
        name: buildah
      params:
        - name: IMAGE
          value: image-registry.openshift-image-registry.svc:5000/cosa199212-dev/jhipster-gateway:latest
      workspaces:
        - name: source
          workspace: shared-workspace
----

Pipeline Execution:

[source,bash]
----
oc create -f pipeline.yaml
oc create -f pipelinerun.yaml
oc get pipelineruns
----

Capabilities Demonstrated:

* Kubernetes-native CI/CD automation
* Container build using Buildah
* Registry authentication via ServiceAccount
* Event-driven deployment rollout
